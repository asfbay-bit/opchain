opchain · mobile demo previews

Five directions for the mobile workbench. Pick one.

The current /demo page is a 4-column desktop IDE that gets cramped on phones. These five directions each compact the same six replay scenarios into a thumb-first shape — same data, different metaphor. Open each on a phone (or in a 390-wide DevTools frame) and see which feels right.

5 directions · same data · standalone HTML · no build step

A
Bottom-Tab IDE
the workbench, but thumb-first
Native-app tab bar. The 4 desktop panels become 4 swipeable tabs: Scenarios · Stream · I/O · Inspector. Topbar keeps the scenario chip visible.
Trade-off Preserves all desktop info, learnable. Loses side-by-side context.
Open A →
B
Reels / Vertical Snap
scroll the replay like a story
TikTok-shaped vertical-snap cards. Each beat = one snap card with the user prompt + Claude reply + artifact chip. Horizontal pill carousel switches scenarios.
Trade-off Cinematic, brand-on. Loses random-access scanning.
Open B →
C
iMessage Replay
watch Claude work, conversational
Auto-replaying chat thread. User bubbles right (ember), Claude bubbles left, beats centered as system dividers. Sticky play/pause/scrub footer.
Trade-off Lowest friction, brand-true voice. Auto-play timing needs care.
Open C →
D
Pipeline Timeline
the build log as a hike
Vertical step-tracker. Beats are stations on a left-edge rail; tap one to expand the user/Claude exchanges beneath. Inputs above, outputs grid below.
Trade-off Scannable, respects long transcripts. Less "live IDE" feel.
Open D →
E
Stacked Workbench
same panels, just stacked
Every desktop panel becomes a <details> accordion with a sticky mini-toc. Scenario picker → Inputs → Transcript (open) → Outputs → Inspector.
Trade-off Smallest engineering lift, fully a11y-native. Least distinctive.
Open E →

Each preview is standalone HTML — open at ./a.html through ./e.html. Real scenario data lives in ./_shared/data.js; tokens in ./_shared/tokens.css. After you pick a direction I'll build the production component at site/src/components/MobileWorkbench.astro and wire it into /demo via a CSS-breakpoint swap with the existing desktop workbench.
