Replays — 5 layout approaches for 1.4

Five ways to keep scenario detail visible without blocking the chat. Each frame shows the same scenario (Audit gate before deploy) and the same chat turn so they're easy to compare. Pick the one you want me to ship; tell me to mix-and-match if you want; or send me back to the drawing board.

A · side-by-side

Detail left, chat right

Two columns at desktop widths (≈38% / 62%). Both fully visible at all times. Stacks vertically below 720px so mobile reads like the current layout. Pros: no toggling, both panels always there. Cons: chat is narrower; long reference scenarios with big tables / diagrams may feel cramped.

B · picture-in-picture

Detail floats over the chat (top-right)

Detail card pinned to the top-right corner of the chat as a compact floating card. Drag to move, click × to dismiss; chat fills full width underneath. Pros: chat takes the full canvas. Cons: the float still occludes part of the chat; drag UX needs work for mobile.

C · tabs (recommended)

Scenario / Chat tab switcher

Two tabs above the panel — scenario + chat. The active tab gets the full panel height. Tabs are keyboard-shortcut bound (1 / 2). Cleanest UX, easiest to ship, scales to any viewport. Pros: chat always full height when selected; both panels are first-class. Cons: one click to compare detail vs chat (vs. side-by-side which shows both).

D · compact summary

Pill row above chat

Detail collapses to a single horizontal pill row above the chat — title, skill pills, runtime, "show details" link. Click the link to open an inline popover with the full description. Chat fills the rest of the panel by default. Pros: minimal vertical real estate, chat dominates. Cons: long titles or many skills wrap awkwardly; details are one extra click away.

E · bottom sheet

Drag-to-reveal detail at the bottom

Detail card lives at the bottom of the chat as a small sheet with a drag handle. Drag up → expand to a full panel; drag down → collapse to a thin strip showing just the title + a peek line. Chat is always full height with the sheet floating over the bottom 30 - 80px depending on state. Pros: chat stays primary; detail is a glance away. Cons: mobile-first interaction; might surprise desktop users; needs tasteful drag mechanics.
