Five different design directions for the redesigned /demo page, after the Try-It / Live surface was removed. Same scenario (audit-gate-before-deploy) so they're easy to compare apples-to-apples. Pick one (or a mash-up) — I'll build it.

Aeditorial · docs

Three-pane docs reading room

Stripe / Linear / Vercel docs aesthetic. Left rail = scenarios + on-this-page nav. Center = wide reading transcript. Right rail = inputs / outputs / skills inspector. Content-first; production-ready.

Beditorial · poster

Magazine cover & 6-up scene grid

WIRED / Apple keynote vibe. Big serif typography poster as the hero, then a 6-up grid of scenario "covers", then a deck-style transcript. Wow-factor; scrolls like an issue.

→
Ctool · IDE

Workbench / IDE chrome

VS Code-style activity rail + sidebar (file tree of scenarios) + tabbed editor (transcript) + inspector + status bar. Looks like a real dev tool. Pro-user feel; dense + utilitarian.

→
Dplayful · interactive

Scrubbable timeline + draggable cards

Game / playground feel. Carousel of scenario cards (hover lifts + tilts), scrubbable timeline with markers, transcript turns with hover-reveal annotations, draggable input/output chips. Highest delight; least docs-y.

→
Ehybrid · wow

Cinematic intro → workbench reveal

Best of B + C. Full-bleed magazine-cover hero (fixed 100vh), scrolls into a workbench layout (scenarios / transcript / inspector). Tour pills pinned bottom. Combines wow-factor entry with pro-user inspection underneath.
