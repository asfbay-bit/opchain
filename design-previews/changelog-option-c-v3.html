v1.4.2Complete bundles, honest checkpointsJun 05, 2026

Emergency patch: full skill trees, oc- aliases, unified checkpoint schema, automated drift detection.

emergency patch18 skillsback-compatible

An out-of-band emergency patch: one user-facing breakage and a long-overdue correctness pass, shipped ahead of v1.5. The breakage — the "download all" skill bundle was shipping only each skill's SKILL.md, silently dropping the references/, scripts/, and examples/ that make a skill actually run.

What's new
  • "Download all" ships complete skills again. opchain-skills.zip now bundles full skill trees, matching the per-skill downloads on /skills.
  • oc- canonical skill names. Every skill gains an oc-<name> alias so opchain skills are unambiguous next to same-named local skills.
  • The orchestrator stops misreporting itself. Persistence docs rewritten to the git-tracked checkpoint it actually uses. The phantom checkpoint.sh is replaced by scripts/checkpoint.mjs.
  • One checkpoint schema, actually enforced. Validator, spec, and README now agree on a single wire schema. Timestamp-sanity and referential checks added.
  • Drift detection. Automated drift detection; stale checkpoints can no longer tell future-you to redo shipped work.
New CLI commands
  • checkpoint:doctor — cross-checks checkpoints against git log + live /api/health, flags drift
  • checkpoint:next — "what should I do next" for the local project
  • checkpoint:status --brief / --since=<date>
  • checkpoint:init — deterministic cold-start scaffold
Back-compatible with v1.4. The oc- names are additive aliases. All 18 skills lockstep-bump to 1.4.2.

v1.7Pick one distribution playlean: marketplace + templates

By v1.6, opchain has 24 skills and AI coverage. The biggest remaining gap is distribution. v1.7 promotes one growth play into a flagship theme.

plannedest. Sep 2026

By the time v1.6 ships, opchain has 24 skills, instrumented pipelines, and AI-native coverage. The biggest remaining gap isn't features — it's distribution. v1.7 deliberately promotes one growth play into a flagship theme instead of accreting more skills. The current lean is marketplace + templates.

The lean plan
  • marketplace. A site surface for community-contributed skills and packs. Compounds once v1.6's telemetry exists to grade community contributions on real usage.
  • template-ops — /template. Opinionated project starters that pre-wire an entire pipeline — stack, skills, and checkpoints — so a new project is ready to run from the first commit.
Alternatives on the table
  • Agency play. monorepo-ops for real multi-project checkpoint scoping, a /for-agencies expansion, and a client-handoff demo scenario.
  • Pipeline depth. discovery-ops (JTBD / opportunity trees, upstream of app-architect) plus qa-ops splitting test-pyramid design out of bug-check.
Lean, not locked — the final v1.7 theme is chosen after the v1.5 / v1.6 retro.
