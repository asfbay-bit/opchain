architecture

How opchain skills chain.

Every opchain skill owns one phase of the development pipeline and communicates with the others through a single JSON file at .checkpoints/<skill>.checkpoint.json. There is no central orchestrator, no message bus, and no framework — each skill is a self-contained prompt package that reads its upstream checkpoints and writes its own.

The pipeline

reverse-spec ──▶ app-architect ──▶ git-ops ──▶ deploy-ops
                      │
                      ├── auto-invokes stack-forge   (Phase 2)
                      ├── design pipeline            (Phase 3, ux-engineer)
                      ├── build loop                 (Phase 6, Generator/Evaluator)
                      └── launch handoff             (Phase 7)

code-auditor ────▶ runs at any stage, gates deploy
integrations-engineer ────▶ runs when external APIs needed
scale-ops ────▶ advisory, no pipeline chain

Checkpoint protocol

Every skill adopts a shared JSON schema for session persistence. The canonical top-level keys are phase, progress_table, context_primer, blockers, and next_actions. When a skill activates it reads its own checkpoint (offering to resume) and checks for upstream checkpoints to pre-load context.

download bundle

Every skill is a single SKILL.md file. Install all ten or just the ones you want. No API keys, no package manager, no build step.

Claude Code CLI

Drop the files into your project's .claude/skills/ directory. Claude Code picks them up on the next session.

curl -L https://opchain.dev/opchain-skills.zip -o opchain-skills.zip
unzip opchain-skills.zip -d .claude/skills/
claude
> /discover

Team (check into git)

git add .claude/skills/
git commit -m "chore: add opchain skills"
git push
