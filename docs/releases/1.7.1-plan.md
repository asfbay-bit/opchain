# opchain 1.7.1 - Skill Install Portability Patch

> **Status:** PATCH RELEASE CANDIDATE.
> **Produced by:** `oc-release-ops`
> **Date:** 2026-07-05
> **Source branch:** `claude/pensive-hawking-1775f3`
> **Source PR:** #376

v1.7.1 releases the pensive-hawking portability fixes as a patch over the
current shipped v1.7 line. It does not introduce a new product surface, new
commands, or a new release theme. It makes the skill bundles that already ship
install cleanly for users outside this repository.

## Why This Is Patch

- The change fixes checkpoint writing and cross-skill protocol loading for
  installed skill bundles.
- It adds catalog/build guards so missing protocol files and broken
  `references/...` citations fail before release.
- It updates the staged v1.8 skills only so their bundled protocol files do not
  reintroduce the same portability defect.
- It is backward-compatible with existing checkpoint JSON and skill commands.

## What Ships

| Area | Patch behavior |
|---|---|
| Checkpoint writes | Installed skills can write `.checkpoints/*.checkpoint.json` directly with file tools instead of depending on repo-only `scripts/checkpoint.mjs`. |
| Shared protocol loading | Every skill bundle carries `references/orchestrator.md` and the bootstrap line needed to read it. |
| Build guards | `gen-skills-catalog.mjs` rejects missing protocol files, missing bootstraps, dangling `references/...` citations, and overlong descriptions. |
| Bundle sync | `sync-skill-bundles.mjs` creates missing bundled references and checks them in CI instead of silently skipping absent files. |
| Audit evidence | `docs/audits/2026-07-04-portability-audit.md` records the portability findings and verification trail. |

## Compatibility

Back-compatible with v1.7. No environment variables, storage migrations, route
changes, or user-facing command changes are required.

Existing checkpoint JSON remains valid. The patch changes how skill bundles
instruct the model to write checkpoint files on machines that do not have the
opchain repo CLI.

## Release Notes Draft

- **Installed skills can persist checkpoints.** The protocol now permits direct
  file-tool writes and treats the repo CLI as an optional in-repo fast path.
- **Every bundle carries the shared protocol.** Missing `orchestrator.md` and
  `checkpoint-protocol.md` references are created and guarded.
- **Catalog validation now catches portability drift.** Missing bootstraps,
  dangling reference citations, and overlong descriptions fail the build before
  a bundle ships.
- **The audit is durable.** The portability audit lives in `docs/audits/` so the
  fix has a reviewable evidence trail.
