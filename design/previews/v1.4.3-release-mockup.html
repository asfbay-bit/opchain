opchain
skills that ship.

a skillchain and checkpoint protocol designed to get you from concept to production to ops

If you've ever tried to build something real with Claude and watched it lose the plot between chats — forgetting your spec, redesigning what it already designed, repeating questions it asked an hour ago — opchain is what you've been looking for.

A set of skills plus a checkpoint protocol — native in Claude Code, and now in Codex or any MCP agent via a hosted endpoint. Each skill specializes in one piece of the puzzle (discovery, spec, design, build, audit, ship, monitor, migrate, scale). A central orchestrator routes between them so you never have to think about which skill to call.

Install them, trigger by name, keep going. No API keys. No SaaS. No vendor lock-in — every skill is a single Markdown file.

install

Four flows.

Every skill is a single SKILL.md file. Install the full set or just the ones you want. No API keys, no package manager, no build step.

⬇ download bundle

01

⌁ Claude Code CLI

Drop the skills into .claude/skills/. Claude Code auto-discovers them on the next session.

# from your project root
curl -L https://opchain.dev/opchain-skills.zip -o opchain-skills.zip
unzip opchain-skills.zip -d .claude/skills/
02

▤ Claude.ai / Claude Desktop

Download the bundle or individual SKILL.md files, then Settings → Customize → Skills → Upload skill.

03

⎇ Team (check into git)

Commit the skills directory so every teammate uses the same prompt pipeline.

git add .claude/skills/ && git commit -m "add opchain skills"
04

⌁ Codex / any MCP agent NEW · v1.4.3

Codex — and any MCP-aware agent (Claude Desktop, Cursor, Windsurf) — reaches opchain over the Model Context Protocol. Point your client at the hosted endpoint and you get the catalog, intent routing, the orchestrator protocol, and checkpoints — the same pipeline Claude Code auto-discovers.

Hosted MCP server — nothing to install:

# ~/.codex/config.toml
[mcp_servers.opchain]
url = "https://opchain.dev/mcp"

Or drop the skills in directly — Codex Agent Skills use the same SKILL.md format:

curl -L https://opchain.dev/opchain-skills.zip -o opchain-skills.zip
unzip opchain-skills.zip -d .codex/skills/
codex

Verify it worked. Run codex and ask "build me an app" — opchain's route tool hands you to oc-app-architect's discovery interview. With the MCP server, the /oc-* commands also appear as prompts in the slash menu.

v1.4.3 · the codex release

One pipeline. Every agent.

opchain was born in Claude Code. As of v1.4.3 the exact same concept-to-ship pipeline runs anywhere the Model Context Protocol does — starting with OpenAI's Codex.

"The skills didn't change. Their reach did. Same 18-skill pipeline, now one config line away from any MCP-aware agent."

Hosted, zero-install

One line in ~/.codex/config.toml points Codex at opchain.dev/mcp. No package to vendor, no key to manage, no build step.

Tools, prompts & resources

The MCP server exposes the skill catalog and intent routing as tools, every /oc-* verb as a prompt, and the orchestrator protocol + checkpoints as resources.

Checkpoints cross sessions

The same JSON checkpoint contract Claude Code uses persists across MCP sessions — start in Claude Code, resume in Codex, never lose the plot.

Or run it offline

An air-gapped mcp/local-server.mjs stdio server ships in the bundle — same catalog, no network.
