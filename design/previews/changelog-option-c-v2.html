v1.4.2 Complete bundles, honest checkpoints latest Jun 05, 2026

Emergency patch: full skill trees in the bundle, oc- aliases, unified checkpoint schema, automated drift detection.

emergency patch 18 skills back-compatible

An out-of-band emergency patch ahead of v1.5: one user-facing breakage and a long-overdue correctness pass. The breakage — the "download all" bundle was shipping only each skill's SKILL.md, silently dropping references/, scripts/, and examples/. The correctness pass unifies the checkpoint schema that the validator, spec, and README had quietly diverged on.

What's new
  • "Download all" ships complete skills again. opchain-skills.zip now bundles full skill trees matching the per-skill downloads on /skills. The previous version was silently delivering broken installs.
  • oc- canonical skill names. Every skill gains an oc-<name> alias — oc-app-architect, oc-git-ops, oc-deploy-ops — so opchain skills are unambiguous next to same-named local skills. Bare names still resolve.
  • Orchestrator stops misreporting itself. Persistence docs rewritten to the git-tracked .checkpoints/orchestrator.checkpoint.json it actually uses. Phantom checkpoint.sh replaced by scripts/checkpoint.mjs.
  • One checkpoint schema, actually enforced. Validator, spec, and README now agree on a single wire format. next_actions required while in-progress. Timestamp-sanity and referential checks added to the validator.
  • Drift detection. Automated drift detection added to orchestrator; stale checkpoints can no longer instruct you to redo already-shipped work.
New CLI commands
  • checkpoint:doctor — cross-checks checkpoints against git log + live /api/health, flags drift
  • checkpoint:next — "what should I do next?" for the local project, no registry needed
  • checkpoint:status --brief / --since=<date> — terse resume view + momentum digest with stale-age flag
  • checkpoint:init — deterministic cold-start scaffold
  • checkpoint -- done <skill> — pop top next_action into recently_done log
Back-compatible with v1.4. The oc- names are additive aliases; nothing renames. Checkpoint wire schema stays pinned at protocol_version 1.0. All 18 skills lockstep-bump to 1.4.2.

v1.5.0
Build the AI app

v1.5 closes the biggest categorical gap in the catalog: nothing today owns AI-feature design. Four new AI-native skills ship wired through the existing pipeline — model routing, RAG, agent harnesses, and prompts as first-class version-controlled artifacts.

Four new skills shipping in v1.5
claude-api · /claude-api

Model routing (Opus/Sonnet/Haiku), prompt caching by default, streaming vs. batch, tool-use patterns, and migration playbooks for model-version bumps. Cost guardrails cap tokens per phase.

rag-forge · /rag

Tri-agent (Designer / Builder / Evaluator). Vector-DB decision tree, embedding-model choice, chunking strategy, hybrid search, and retrieval eval with golden sets + recall@k / NDCG.

agent-forge · /agent

Tri-agent (Planner / Builder / Evaluator). Claude Agent SDK scaffold, subagent topology, tool-budget design, harness-loop shapes (chat / autonomous / HITL), and eval over golden trajectories.

prompt-ops · /prompt

Prompts as first-class artifacts: file-shaped versioning, eval datasets, regression detection. /prompt regress runs the golden set on PR-changed prompts and blocks on a drop.

Cross-skill ripples
  • app-architect — gains an "AI app?" decision branch in /discover that routes to the four new skills.
  • stack-forge — vector-DB packs added: pgvector, Turbopuffer, Pinecone, Supabase Vectors.
  • code-auditor — new rules: prompt-injection detection, tool-use boundary safety.
  • bug-check — anti-patterns for common LLM mistakes: empty system prompts, missing tool-use validation, unbounded context windows.
  • /demo worker — rewritten end-to-end on claude-api patterns to dogfood the skill.
GTM alongside v1.5

New /ai-recipes walkthroughs, "Build AI apps" row on /compare, AI-app variant of /pipeline-builder, a /blog dev-log scaffold with launch posts, and a /for-agencies page. Newsletter flag wired.

Back-compatible with v1.4. All 22 skills version-bump to 1.5.0. Four new skills ship gated behind their skills.registry.<id>.enabled flags — flip to enable, no reinstall needed.

v1.6The instrumented pipelineest. Jul 2026

cost-ops + telemetry-ops + checkpoint-protocol v1.3. Answers "what did this feature cost?" and "is my pipeline actually improving?"

planned 2 new skills checkpoint v1.3 ▲ 18 votes
plannedest. Jul 202624 skills total

v1.5 introduces four AI-native skills; the predictable next question is "what did that feature cost to ship, and is my pipeline actually getting better?" v1.6 answers both. Cost attribution per skill phase, opt-in local telemetry, graded evals, and the first checkpoint-protocol bump since v1.2.

New skills
  • cost-ops — /cost. Attributes Anthropic spend to each skill phase. Adds budget gates to checkpoints: if a phase would bust the sprint budget, orchestrator asks before proceeding. Recommends model-tier routing — Haiku for cheap repetitive phases (linting, summarization), Opus for spec and audit. First skill to answer "what did this feature cost to ship?"
  • telemetry-ops — /telemetry. Opt-in, local-first usage metering written to .checkpoints/usage.sqlite. Aggregates anonymized stats on which skills and commands people actually use — the data behind the new public /dashboard. Nothing leaves your machine unless you explicitly opt in and confirm the payload.
Cross-skill changes
  • bug-check + code-auditor stop emitting bare pass/fail and start scoring against a stable rubric (A–F for auditor, 0–100 per check for bug-check). Scores are recorded in the checkpoint, giving you a quality trend over time.
  • monitoring-ops gains an "AI-app" template covering token rate, eval regression drift, and hallucination-rate spikes alongside the standard latency/error alerts.
  • orchestrator's /ops next factors cost and remaining budget into what it recommends — won't suggest code-auditor if you're at 90% of the sprint token budget.
  • release-ops publishes a cost breakdown in the release notes: "v1.6 shipped at an average of $0.42 per feature, down 18% from v1.5."
checkpoint-protocol v1.3

First bump since the v1.2 PM-MCP fields. Adds cost, eval_scores, and telemetry_handle to the wire schema. migration-ops rolls it across every skill in a single PR — the same pattern the governance frontmatter roll-out used in v1.4. The bump is backwards-read-compatible; old checkpoints load fine, new fields default to null.

Back-compatible with v1.5. All 24 skills lockstep-bump to 1.6.0.
