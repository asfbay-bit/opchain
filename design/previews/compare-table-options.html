Three directions for the compare-table styling.

Same data, three different ways to add color and structure so the table reads as a comparison instead of a wall of prose. All three keep the existing ember/cream palette — they differ in how hard they lean on color to do the scanning work.

Option A

Verdict pills above every cell

Each cell leads with a small colored pill that classifies the answer — Yes (green), Partial (amber), No (red), Free / Paid, None / Locked-in. Descriptive text stays below for the reader who wants details, but the eye can scan a column top-to-bottom in 2 seconds and see how it stacks up.

Pros
  • Instant scannability — colored dots align vertically
  • Easy to extend to new platforms; the pill taxonomy is fixed
  • Keeps descriptive nuance intact
Cons
  • Adds visual noise — every cell has a chip on top of text
  • Risk of oversimplifying real "it depends" cases

Option B

Soft heatmap cells + ember-hero opchain column

Every cell gets a faint background wash — green for good, amber for partial, red for missing, blue for "neutral / paid". The opchain column gets a bold ember gradient header and a stronger ember wash that overrides the heatmap, so it visually dominates the table. The page becomes legible from across the room.

Pros
  • Strongest visual signal — pattern emerges immediately
  • opchain column reads as "the protagonist", not just a tinted one
  • No extra chips above text — clean cells
Cons
  • Most visually loud — risks feeling salesy / loaded
  • Heatmap classification is editorial — every cell is a judgment
  • Light-mode equivalents need re-balancing

Option C

Grouped categories, zebra rows, ember "winner" dots

Features cluster into named sections (Architecture · Operations · Cost & Lock-in) with sticky ember-colored category headers. Subtle zebra striping helps the eye track across rows. On each row, a small ember dot marks the column that "wins" that dimension — usually opchain, but not always (Devin actually wins session persistence). Adds structure more than color saturation.

Pros
  • Quietest of the three — feels editorial, not loaded
  • Category groups break the 10-row table into 3 scannable chunks
  • Winner dots stay honest (opchain doesn't always win)
Cons
  • Less instantly readable than B; needs the reader to engage
  • "Winner" framing is editorial too — you're picking one per row
  • Requires regrouping the FEATURES list into named sections
