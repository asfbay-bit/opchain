# Sprints

Per-sprint contracts and evaluation rounds, generated by the `oc-app-architect`
build loop.

| Dir | What's in it |
|---|---|
| `sprint-7a/` | `contract.md` (scope/DoD for the sprint) + `eval-round-1.md` (Generator → Evaluator score) |
| `sprint-7b/` | Same shape as 7a |
| `sprint-7b-baseline/` | `contract.md` only — the baseline contract sprint-7b was diffed against |
| `sprint-7c/` | Same shape as 7a/7b |

## How this relates to `roadmap/` and `specs/`

Three directories cover related but distinct ground:

- **`roadmap/`** — forward-looking plans: what's being built and why, before
  work starts (`00-approach.md` … `09-v16-sprint-plan.md`, `B-10-*`).
- **`sprints/`** (this dir) — execution records: the contract a sprint was
  built against and the evaluator's score once it shipped.
- **`specs/`** — reverse-engineered state of the codebase (what actually
  exists), refreshed periodically via `/rev-full` / `/rev-diff`, plus dated
  audit/gap-analysis snapshots.

Rule of thumb: if you're deciding what to build next, start in `roadmap/`.
If you're checking what a specific sprint's contract required, look here.
If you're checking what the codebase currently does, look in `specs/`.
